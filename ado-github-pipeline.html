🔄 ADO ↔ GitHub Automated Pipeline

End-to-end workflow from backlog item to deployed code

1

ADO Create Backlog Item

A Product Backlog Item is created in Azure DevOps (brandsafway2/create_canvas).

POST /_apis/wit/workitems/$Product%20Backlog%20Item → Work Item ID: AB#42
2

GitHub Issue Created & Linked

A GitHub issue is created with the sync-ado label. The ado-sync.yml workflow links it to the ADO work item via AB# reference.

Issue #2: "Add health check endpoint at GET /api/health" Labels: [copilot, sync-ado] → Comment: ✅ Azure DevOps work item created: AB#42
3

Copilot Agent Assigned

The assign-copilot.yml workflow detects the copilot label and assigns the Copilot Coding Agent to the issue.

Trigger: issues.labeled → "copilot" Action: github.issues.addAssignees(["Copilot"]) → 🤖 Copilot Coding Agent assigned
4

Copilot Creates Branch & Develops

Copilot reads .github/copilot-instructions.md and copilot-setup.yml, creates a branch, writes code and tests following project conventions.

Branch: copilot/issue-2-health-endpoint Files: src/routes/health.ts (new) src/routes/index.ts (modified) tests/health.test.ts (new) Runs: npm ci → npm run build → npm test ✓
5

Copilot Opens Pull Request

The agent creates a PR referencing both the GitHub issue and the ADO work item.

PR #3: "Add health check endpoint" Body: "Closes #2 | Resolves AB#42" Branch: copilot/issue-2-health-endpoint → main
6

CI GitHub Actions Validates

The ci.yml workflow runs lint, build, and tests on Node 18.x and 20.x.

Jobs: build-and-test (matrix: [18.x, 20.x]) Steps: checkout → npm ci → lint ✓ → build ✓ → test ✓ Status: ✅ All checks passed
7

GitHub Review & Merge

Copilot Agent addresses review comments. Once approved, the PR is merged to main.

Review: Approved by reviewer Merge: Squash and merge → main Issue #2: Automatically closed
8

ADO Pipeline Validates Build

The Azure DevOps pipeline (azure-pipelines.yml) triggers on merge, running full build, test, and deploy stages.

Stage: Build & Test → Node.js install → npm ci → lint → build → test → Publish JUnit results Stage: Deploy to Staging → Environment: staging ✓
9

ADO Work Item → Done

The ado-pr-status.yml workflow detects the AB#42 reference in the merged PR and updates the ADO work item to Done.

Trigger: pull_request.closed (merged=true) Match: AB#42 in PR body PATCH: /fields/System.State → "Done" → ✅ Work item AB#42 resolved
