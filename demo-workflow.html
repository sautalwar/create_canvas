Demo Walkthrough — 10 Steps

1 ADO Work Item / Backlog Item Created Azure DevOps
  • Open Azure Boards and create a new Product Backlog Item
  • Title: "Add health check endpoint to the API"
  • Add acceptance criteria and description
  • Show the item on the Kanban board in "New" state
💡 SAY: "This is where every developer workflow starts — a work item in your existing ADO backlog. Nothing changes about how your team plans work."
2 GitHub Issue Linked to ADO GitHub + ADO
  • Option A: Azure Boards GitHub App — auto-links issues and PRs
  • Option B: GitHub Action syncs issue → ADO work item via REST API
  • Option C: AB#1234 syntax in commits/PRs for bidirectional linking
  • Show the GitHub issue with the ADO link in the description
💡 SAY: "Three ways to connect — pick what fits your workflow. The Azure Boards App gives you zero-config bidirectional linking."
3 Copilot Agent Assigned Issue GitHub Copilot
  • Assign the GitHub issue to @copilot
  • Copilot Coding Agent picks up the issue autonomously
  • Agent reads the issue, understands the codebase and conventions
  • Follows .github/copilot-instructions.md for project standards
💡 SAY: "One click — assign to Copilot. It reads your codebase, understands your patterns, and gets to work. No prompting required."
4 Agent Creates Branch GitHub Copilot
  • Copilot automatically creates a copilot/fix-{issue} branch
  • Branch is based on main with latest code
  • Show the new branch appearing in the GitHub UI
💡 SAY: "Standard git workflow — the agent creates an isolated branch just like any developer would."
5 Agent Develops Enhancement GitHub Copilot
  • Copilot reads existing code patterns (src/routes/items.ts)
  • Implements the health check endpoint following conventions
  • Uses ApiResponse<T> wrapper type consistently
  • Registers new route in src/routes/index.ts
💡 SAY: "Notice how the agent follows your existing patterns — same response types, same route structure, same coding style."
6 Agent Commits Code + Tests GitHub Copilot
  • Copilot writes matching test cases in tests/health.test.ts
  • Tests follow the existing Jest + Supertest pattern
  • Commits with descriptive message referencing the issue
  • Includes both implementation and test files
💡 SAY: "Code AND tests — the agent doesn't just implement, it validates its own work with tests that match your existing test patterns."
7 Agent Raises PR GitHub Copilot
  • Copilot opens a draft Pull Request
  • PR description includes AB#1234 to link back to ADO
  • Closes the linked GitHub issue reference
  • ADO work item automatically moves to "In Progress"
💡 SAY: "The PR links back to ADO automatically — your project managers see progress in real-time on their board without leaving ADO."
8 Development Lead Reviews / Approves Human Review
  • Dev lead reviews the PR — code quality, patterns, test coverage
  • Branch protection rules require at least 1 approval
  • Review comments can trigger Copilot to make revisions
  • Human stays in the loop — AI proposes, humans approve
💡 SAY: "This is the governance layer — no code reaches production without human review. The AI accelerates, humans validate."
9 ADO Pipeline Validates Azure Pipelines
  • Azure Pipelines triggers on the PR — build, lint, test
  • GitHub Actions CI also runs (Node 18.x + 20.x matrix)
  • Status checks appear in the PR — both must pass
  • Test results published back to ADO
💡 SAY: "Your existing ADO pipelines validate the AI's work — same quality gates as human-written code. No shortcuts."
10 Merge + Deployment Production
  • Merge the PR after approval + green checks
  • ADO pipeline triggers deployment to staging
  • ADO work item auto-transitions to "Done"
  • Full traceability: work item → issue → branch → PR → deployment
💡 SAY: "From backlog item to production — fully traced, fully validated, fully autonomous. What used to take a sprint takes minutes."

Three Integration Methods

🔗 Azure Boards GitHub App

Install from GitHub Marketplace. Zero-config bidirectional linking between ADO work items and GitHub issues/PRs. Use AB#1234 syntax anywhere.

⚡ GitHub Actions + ADO API

Custom automation via ado-sync.yml workflow. Creates ADO work items when GitHub issues are labeled sync-ado. Full control over the sync logic.

🏷️ AB# Commit Syntax

Include AB#1234 in commit messages or PR descriptions. ADO automatically links and updates work item state. Works with any Git client.
